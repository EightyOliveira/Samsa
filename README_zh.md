### 🌟 核心理念

> **拥抱“简单同步”，放弃“复杂异步”**

无需回调地狱或复杂的响应式编程，即可获得接近异步 I/O 的性能，同时保持同步代码的可读性与调试体验。

---

### ⚡ 核心特性

1. **原生支持虚拟线程**  
   充分利用 Java 21+ 的 Project Loom 能力，以极低开销实现高并发 I/O 处理。

2. **绕过传统 Servlet 容器**  
   放弃 Tomcat、Jetty 等重型容器，显著减少启动开销与内存占用。

3. **极简高性能路由**  
   基于 **Radix Tree** 实现，路由匹配时间复杂度为 O(k)。

4. **零分配上下文对象**  
   请求处理过程中不创建额外包装对象，极致降低 GC 压力。

5. **极致轻量**  
   完美支持 GraalVM Native Image 等轻量级部署方式。
